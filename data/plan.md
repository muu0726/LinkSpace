# LinkSpace 開発計画 (Implementation Plan)

## 1. プロジェクト概要
- **コンセプト**: 放置された土地を地域の資産に変えるプラットフォーム。煩雑なやり取りを排除した「メッセージレス・リクエスト制」を採用。
- **技術スタック**: Next.js (App Router), React, TypeScript, Tailwind CSS, shadcn/ui, Supabase (PostgreSQL, Auth, Storage, Edge Functions)

## 2. システム構成とアーキテクチャ
- **フロントエンド**: Next.js App Routerを使用したサーバーサイドレンダリング(SSR)・クライアントサイドレンダリング(CSR)の最適化。PWA対応によるオフライン利用サポート。
- **バックエンド/DB**: Supabaseを活用し、認証(Auth)、データベース(PostgreSQL)、画像保存(Storage)をサーバーレスで構築。
- **セキュリティ**: Supabase Row Level Security (RLS) によるデータアクセス制御。機密情報は当事者のみ閲覧可能とする。OWASP基準の脆弱性対策、および退会時の適切なデータ削除・匿名化を行う。
- **決済システム**: 独自ポイント通貨(LinkPoints: LP)を使用。DBのRPC（ストアドプロシージャ）と行ロックを用いたセキュアなトランザクション処理。

## 3. 開発フェーズ定義
要件定義をベースに、依存関係を考慮してシステムを5つのフェーズに分割し開発を進めます。

### フェーズ1: 基盤構築・認証 (環境構築とユーザー管理)
プロジェクトの土台となる環境構築と、すべての起点となるユーザー認証を実装します。
- Next.js プロジェクトのセットアップ、Tailwind CSS・shadcn/uiの導入
- PWA対応の基礎設定（アプリアイコンは `img/icon.png` を使用）
- Supabaseプロジェクトの準備とAPI接続設定
- メールアドレス・パスワード認証の実装
- ユーザープロフィールの作成、閲覧、編集、ログアウト機能
- アプリ共通の型定義とAPIレスポンス統一ルール(`ApiResponse`)の適用

### フェーズ2: 物件管理・検索機能 (プラットフォームのコア機能)
貸し手が物件を公開し、借り手がそれを探し出せるようにするコア機能を実装します。
- 物件データモデルの設計（RLS設定を含む）
- 物件の登録、編集、画像アップロード、公開/非公開設定
- カレンダーを利用した貸出可能日・ガイドルールの設定
- LLMを活用した物件説明・利用ガイド草案の自動生成機能（拡張機能）
- エリア検索、タグフィルタリング機能
- 地図表示機能（マップUI上でのマーカー表示）
- お気に入り登録・一覧機能

### フェーズ3: 予約・ポイント決済システム (最重要ビジネスロジック)
本システムの肝である、ポイント(LinkPoints)を通じたエスクロー決済・予約フローを構築します。
- ポイント管理テーブル、予約テーブルの設計
- トランザクション・行ロックを利用した堅牢な決済処理(RPC)の実装
- 初回登録時の初期LP付与（100,000LP）
- 予約リクエストの送信〜オーナーによる承認/却下のワークフロー
- 予約時のポイント引き落とし、完了時の報酬加算
- キャンセルポリシー（3日前100%、前日50%、当日0%）に基づく自動返金ロジック
- ポイント取引履歴の表示

### フェーズ4: 利用ガイド・レビュー・通知・Q&A (体験向上とコミュニケーション)
実際の「利用」やユーザー間の疑問解消にフォーカスし、ユーザー体験を向上させる機能を実装します。
- 予約確定済みの借り手向け、利用ガイド自動生成とオフライン閲覧機能
- 現地でのチェックイン、チェックアウト報告フロー
- 相互レビュー（物件/貸し手に対する評価と、借り手に対する評価）
- 物件詳細画面での公開Q&A（借り手の質問投稿、貸し手の回答投稿、一覧表示）機能 (fun-057, 058, 059)
- 定型アクション時（承認・却下・リクエスト等）のシステム通知機能
- リアルタイム通知（Webプッシュまたはシステム内通知センター）

### フェーズ5: 管理者機能・全体最適化 (運用と品質向上)
システムの安定稼働と運用を支える機能、および最終的なブラッシュアップを行います。
- `role='admin'` のみに許可された `/admin` 管理画面の構築
- 全体統計（ユーザー数、累計LP流通額など）のダッシュボード表示
- ユーザー管理（アカウント利用停止など）および物件検閲機能
- 管理者による手動ポイント調整ロジック、エラーログ監視機能
- スケルトンローディングの最適化（3秒以内表示の達成）
- ダークモード/ライトモードの対応、アクセシビリティ（WCAG 2.1準拠）の最終確認
